# SmartRoute 商业演示脚本 (Demo Script)

> **场景**: 某大件运输公司接到一单从 **福州** 到 **厦门** 的变压器运输任务。
> **目标**: 展示 SmartRoute 如何帮助调度员快速选线、预估成本并规避风险。
> **演示时长**: 5-8 分钟

---

## 1. 开场引入 (1分钟)

**话术**:
"各位好，今天为大家演示 SmartRoute 系统。大家知道，传统的大件运输选线，调度员需要对着地图手工画线，还需要去查限行政策，算油耗，非常繁琐。现在，让我们看看 SmartRoute 如何在 1 分钟内完成这些工作。"

**操作**:
- 打开 SmartRoute 首页。
- 强调界面简洁专业，左侧是操作区，右侧是地图区。

---

## 2. 车辆资质录入 (2分钟)

**核心卖点**: **专业级车辆建模能力** (展示我们懂业务)

**操作**:
1.  点击左侧表单的 **"管理通行资质"** 按钮。
2.  **展示车辆信息**:
    -   **车牌号**: 输入 `闽A-88888`。
    -   **轴组配置**: 重点演示这里。输入 `[1, 2, 3]` (表示1轴+2轴+3轴)。
    -   **轴重/轴距**: 随意输入一些数据，强调系统支持录入这些GB1589关键参数。
3.  **展示货物信息**:
    -   输入货物名称 `大型变压器`。
    -   输入长宽高 `10m x 3m x 3.5m`。
4.  点击 **"保存"**，看到提示 "保存成功"。
    -   *话术*: "大家看，这些数据一旦录入，就会保存在本地数据库中，下次直接调用，不用重复输入。"

---

## 3. 智能选线演示 (2分钟)

**核心卖点**: **多策略并发 + 智能去重** (展示技术实力)

**操作**:
1.  回到主界面。
2.  **起终点输入**:
    -   起点: 输入 `福州` (选择下拉提示中的地点)。
    -   终点: 输入 `厦门`。
3.  **出发时间**:
    -   选择 **"明天凌晨 02:30"**。
    -   *话术*: "注意，我故意选了一个凌晨的时间，看看系统会不会通过。"
4.  **备选路线数**: 滑动滑块选择 `3` 条。
5.  点击 **"开始规划"** 按钮。

**观察点 (Wait for it)**:
-   强调速度: "大家看，不到 2 秒钟，系统并发计算了速度优先、费用优先等多条策略。"
-   **结果展示**:
    -   看到地图上绘制了多条路线。
    -   左侧结果面板出现了 3 张卡片。

---

## 4. 结果分析与决策 (2分钟)

**核心卖点**: **TCO 成本模型 + 安全预警** (展示商业价值)

**操作**:
1.  **夜间预警**:
    -   指向结果面板顶部的 **红色/橙色警告条**。
    -   *话术*: "系统敏锐地发现了出发时间是凌晨 2 点，自动弹出了疲劳驾驶和夜间禁行预警。这在传统导航里是很容易被忽略的。"
2.  **成本对比**:
    -   展开第一条路线（推荐路线）。
    -   指着 **"预估总费用"**: "系统不仅算了过路费，还根据我们的重卡油耗模型算出了油费。"
    -   对比第二条路线（比如费用优先）："看这条路，虽然慢了点，但是路费省了 200 块。调度员可以根据急不急来选。"
3.  **NLP 摘要**:
    -   指着路线描述："大家看这个描述，'沿G15沈海高速途径...'，这是我们用 NLP 技术自动生成的，非常符合司机的阅读习惯。"

---

## 5. 数据导出与总结 (1分钟)

**核心卖点**: **数据闭环**

**操作**:
1.  点击结果面板右上角的 **"导出数据 (JSON)"** 按钮。
2.  展示下载的文件。
    -   *话术*: "选好线后，一键导出。这个文件包含了刚才的路线坐标和车辆信息，可以直接发给下游的桥梁验算团队，或者导入到司机的 APP 里。"

**结束语**:
"这就是 SmartRoute，用算法替代人工，用数据驱动决策。谢谢大家！"
