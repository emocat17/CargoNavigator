第四部分: 系统级服务与治理

4.1 身份、访问与权限管理

描述: 标准的IAM服务，负责用户认证和授权。

  • RBAC: 基于角色的访问控制（司机、规划员、合规官等）。
  • SSO: 支持与企业现有身份提供商集成。
4.2 合规与审计追踪引擎

描述: 系统的“黑匣子”，为每次规划创建不可篡改的完整记录，对风险管理至关重要。

  • 不可变日志: 记录查询、上下文、算法、决策等所有信息。
  • 可追溯性: 允许事后完整重构任何决策链条。
4.3 模型与算法生命周期管理

描述: 一套MLOps/AIOps工具和流程，确保系统智能核心的持续迭代与优化。

  • LLM微调流水线: 自动化地使用新数据再训练模型。
  • 算法版本控制: 提供算法的中央仓库，支持A/B测试和回滚。
  • 反馈闭环: 将真实运营反馈纳入再训练数据集中。

第三部分: 用户应用与交互层

3.1 司机交互模块

描述: 司机与系统交互的主要触点，安装于移动或车载终端，强调简洁易用。

  • 对话式交互: 通过自然语言发起路线规划。
  • 路线任务简报: 可视化呈现路线，高亮关键节点。
  • 合规路径导航: 锁定已审批路线，确保合规。
  • 在途问答: 实时查询前方路况约束。
3.2 管理决策支持驾驶舱

描述: 面向规划员、调度员的Web应用，是系统的指挥控制中心。

  • 场景分析: 详尽参数规划，多方案对比。
  • 交互式可视化: “路网运行一张图”，叠加显示各类约束。
  • “假设”情景规划器: 模拟调整参数，辅助决策。
  • 合规审查工作流: 自动生成合规清单并提供LLM解释。
3.3 可视化与报告服务

描述: 为所有前端应用生成视觉元素和文本报告的后端服务。

  • 地图渲染: 生成底图瓦片与动态数据层。
  • 报告生成: 生成PDF格式的合规与存档报告。
  • 仪表盘分析: 提供运营统计数据图表。

第二部分: 核心智能与编排引擎

2.1 LLM智能体核心

描述: 系统的中心编排器，负责理解、推理和任务分解，而非直接计算。

  • 自然语言理解: 解析司机与管理员的复杂请求。
  • 意图识别: 准确提取请求中的关键参数。
  • 工具/函数调用: 核心功能，按逻辑顺序调用下游服务（RAG、算法、可视化）。
2.2 RAG服务

描述: LLM与知识库的桥梁，将通用智能与领域知识结合。

  • 查询转换: 将LLM请求转为数据库查询指令。
  • 混合检索: 并行查询向量与结构化数据库。
  • 提示词注入: 将检索到的事实动态注入LLM提示词，确保决策基于事实。
2.3 多算法服务总线

描述: API网关，解耦推理引擎与计算引擎，实现模块化和可扩展性。

  • 服务发现: 感知算法套件中的可用算法。
  • 请求路由: 将请求精确路由到合适的算法。
  • API版本管理: 允许算法独立更新而不影响LLM。
2.4 路径规划算法套件

描述: 独立的计算引擎微服务集合，作为LLM的“工具箱”。

  • 分层式廊道搜索: 宏观层面快速确定可行廊道。
  • 基于约束的A*: 高效的单目标（最快/最低成本）路径搜索。
  • 多目标优化: 平衡时间、成本、风险等多个冲突目标。
  • 车辆动力学仿真: 精确判断复杂节点的通过性。
  • 反事实建议引擎: 在无解时提出修改参数的建议。

第一部分: 基础数据与知识核心

1.1 统一路网图谱

描述: 物理路网的数字孪生体，一个富含拓扑关系的图数据库，是路径规划的基础。

  • 节点: 交叉口、桥梁、隧道、收费站等，含详细属性。
  • 边: 连接节点的道路分段，含静态（长度、等级）和动态（限速、拥堵）属性。
1.2 多模态约束知识库

描述: LLM的“长期记忆”，结合向量数据库和结构化数据库，为RAG提供基板。

  • 结构化数据: 桥梁限重、隧道净高等精确数值。
  • 非结构化数据: 施工报告、气象预警、历史事件等文本信息。
1.3 动态数据采集与融合服务

描述: 核心数据管道，负责从外部来源收集、清洗、融合实时数据，保持系统“鲜活”。

  • 数据来源: 交通部门API、GNSS、气象服务、车辆遥测等。
  • 功能: ETL处理、数据融合、实时警报推送。

三大核心理念

深入理解驱动系统运行的基础原则。

1. 数据驱动决策

系统决策的基石是全面、精准、实时的多维数据。我们将物理世界数字化，为智能计算提供坚实的真理之源。

2. 智能编排引擎

我们不让LLM直接计算，而是发挥其理解与推理的优势，作为“指挥官”编排调用最合适的专业算法工具来解决问题。
