# CargoNavigator 前端重设计 设计文档

**日期**: 2026-06-17
**状态**: 已批准 → 待实现

---

## 1. 问题诊断

当前布局问题：
- 左侧 420px 抽屉塞了 5 个 tab（路径规划/智能问答/运输追踪/护送监控/数字档案），地图被挤在右侧
- 用户核心操作流程（输入起终点 → 查看路线 → 评估安全 → 许可证 → 监控 → 档案）被分散到5个互不相通的tab
- 空间利用极差——地图是核心但只占了约60%宽度

## 2. 新设计：地图为中心的3页导航

### 2.1 整体布局

```
┌──────────────────────────────────────────────────────┐
│  🚛 CargoNavigator    [🗺 路线规划] [🚚 运输管理] [📦 数字档案] │  ← 顶部导航栏
├──────────────┬───────────────────────────────────────┤
│  浮动面板     │        全屏 AMap 地图                  │
│  (可收起)    │    · 路线高亮（多色）                   │
│              │    · 桥梁标记+评级颜色                  │
│              │    · 施工路段覆盖                       │
│              │    · 右下角 AI 助手浮球                 │
├──────────────┴───────────────────────────────────────┤
│  🟢 桥梁:8座  🟡 施工:3处  🔴 限速:60km/h            │  ← 底部状态栏
└──────────────────────────────────────────────────────┘
```

### 2.2 页面1：路线规划（默认主页）

**浮动面板**（左上角，300px宽，可收起）：
- 起点输入框（地址or坐标，带地图选点按钮）
- 终点输入框
- 车辆快捷配置（展开/收起）：长/宽/高/总重/轴数
- [开始规划] 按钮
- 结果区：路线卡片列表（方案1/2/3），每条显示 距离/耗时/过路费
- 选中路线后出现 [评估安全] [生成许可证] 按钮

**地图**（全屏）：
- 渲染所有路线（不同颜色）
- 桥梁标记（绿色=安全，橙色=关注，红色=风险）
- 施工路段红色半透明覆盖
- 点击路线卡片 → 地图聚焦该路线

**AI 助手浮球**（右下角）：
- 点击展开为对话面板（右下角 380x500 浮动窗口）
- SSE 流式回答
- 自动带入当前路线上下文
- 结构化评估卡片（风险等级/桥梁/施工/尺寸）

**底部状态栏**：
- 选中路线的摘要：桥梁数/施工数/限速/预估费用/风险等级

### 2.3 页面2：运输管理

**布局**：左侧订单列表 + 右侧详情/地图

**订单列表**（320px宽）：
- 顶部搜索/筛选：状态筛选（待审批/已发证/运输中/已完成）
- 订单卡片：单号、路线摘要、状态标签、时间
- 点击选中

**右侧面板**：
- 上方：订单详情卡片（状态时间线、车辆信息、路线摘要、附件）
- 下方（=50%高度）：地图
  - 已发证订单：显示计划路线
  - 运输中订单：启动实时监控 → SSE GPS 追踪 + 检查点通过 + 告警
  - 已完成订单：轨迹回放

**操作按钮**：
- [开始运输] / [启动监控] / [停止监控] / [查看档案]

### 2.4 页面3：数字档案

**布局**：左侧档案列表 + 右侧详情

**档案列表**（320px）：
- 已完成订单列表（搜索+筛选）
- 点击选中

**右侧详情**：
- 摘要卡片：GPS轨迹点数 / 平均速度 / 最高速度 / 行驶时长
- Tab切换：事件时间线 | 轨迹回放 | 检查点明细
- 轨迹回放：地图 + 可调速播放（1x/2x/5x/10x）
- 导出按钮：[导出JSON] [导出PDF]
- 检查点表格：桩号/类型/通过时间/延误

## 3. 数据流

```
用户输入起终点 → POST /routes/plan → 地图渲染路线
    → 用户选路线 → POST /routes/assess → 桥梁+施工数据绑定到地图标记
    → AI浮球对话 → POST /agent/chat (SSE) → 结构化评估卡片
    → 生成许可证 → POST /permit/generate
    → 创建运输单 → POST /tracking/orders
    → 切换到"运输管理" → 启动监控 SSE → GPS实时追踪
    → 停止 → 数据归档 → 切换到"数字档案" → 查看+导出
```

## 5. 路由/导航

不引入 vue-router，使用 Quasar 的 v-model tab 切换 + 简单的组件切换。

```javascript
// App.vue — 三个顶层页面
const pages = ['planner', 'transport', 'archive']
const currentPage = ref('planner')
```

顶部导航栏：3个按钮切换页面。

## 6. 样式主题

- 保持 Quasar 默认主题（Material Design）
- 地图面板浮动使用 `position: absolute` + `z-index`
- 浮动面板有半透明背景 + backdrop-filter 模糊
- AI 浮球：右下角固定，圆形按钮，呼吸动画
- 底部状态栏：固定高度 40px，半透明深色背景

## 7. 测试策略

- 页面切换不丢失状态（路线数据跨页面共享）
- 地图组件在页面切换时正确初始化/销毁
- AI 浮球在所有页面可用
- SSE 连接在页面离开时正确断开
