
### 2.5 路线描述与标签优化 (Description & Labels)
- **路线描述优化**:
  - **后端**: 
    - `process_route_data` 中增加了隧道名称过滤逻辑，避免描述过长。
    - 实现了 **高级去重算法** (`deduplicate_redundant_points`)，涵盖：
      - **"沿" 前缀合并**: 如 "沿吉口互通--吉口互通" 简化为 "吉口互通"。
      - **包含关系简化**: 如 "沿G1517...途径畔溪互通--G1517..." 简化为 "G1517..."。
      - **跨节点去重**: 在经过互通/枢纽等节点后，若道路未变，不再重复显示道路名称 (如 "G1517--枢纽--G1517" 简化为 "G1517--枢纽")。
      - **主辅路去重**: 自动隐藏紧随主路出现的辅路 (如 "金井湾大道--金井湾大道辅路" 简化为 "金井湾大道")。
    - 强制使用用户输入的起点/终点名称替换 API 返回的 "起点"/"终点" 占位符。
  - **前端**:
    - 优化了费用和距离的标签显示，明确区分 "预估总费用" (含油费+路费) 与 "仅路费"，以及 "总里程" 与 "隧道总长"。
- **出发时间支持**:
  - **后端**: 增加了 `departure_time` 参数支持，用于更精准的夜间行车判定。
  - **前端**: 在规划表单中增加了出发时间选择器，默认当前时间。
- **验收**: 已通过 `Test/verify_route_description.py` 验证描述生成的简洁性和标签准确性。
