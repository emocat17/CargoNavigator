# 项目共识文档：大型货车智能选线系统

## 1. 核心共识

### 1.1 项目定义
本项目是一个前后端分离的Web应用，旨在为大型货车提供基于高德地图的智能路线规划服务。系统将根据用户输入的起终点和车辆物理参数（长宽高重），计算推荐路线并在地图上可视化展示。

### 1.2 验收标准
1.  **界面交互**：
    - 用户能在前端界面输入起点、终点（支持模糊搜索或地址输入）。
    - 用户能输入车辆参数（长、宽、高、总重、轴重）。
2.  **路线计算**：
    - 后端成功调用高德地图API。
    - 返回至少一条可行路线（包含距离、预计时间）。
3.  **地图展示**：
    - 前端正确渲染高德地图。
    - 在地图上绘制起点、终点标记。
    - 在地图上绘制规划出的路线轨迹。
4.  **工程质量**：
    - 前端使用 Vue 3 + Quasar。
    - 后端使用 Python FastAPI。
    - 代码结构清晰，无桥梁验算冗余代码。

## 2. 技术规格

### 2.1 架构模式
- **Client-Server**: RESTful API 通信。
- **Frontend**: Single Page Application (SPA).
- **Backend**: Micro-framework based web service.

### 2.2 数据协议
- **请求格式**: JSON
- **响应格式**: JSON (标准封装 `{code: 200, data: ..., msg: ""}`)

### 2.3 环境依赖
- **Node.js**: >= 16 (前端构建)
- **Python**: >= 3.10 (后端运行)
- **Amap Key**: 需复用或新申请高德地图Key。

## 3. 风险与约束
- **API配额与权限**: 高德地图货车路径规划API可能需要企业权限。如果当前Key不支持，将降级使用标准驾车策略，但保留参数传递接口以便未来升级。
- **数据准确性**: 路线规划结果完全依赖高德API返回的数据。
