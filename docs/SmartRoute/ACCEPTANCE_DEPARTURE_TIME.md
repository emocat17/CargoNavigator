# 验收报告：预计出发时间 (Departure Time)

## 1. 任务概述
在路线规划中新增 "预计出发时间" 属性，允许用户设置未来的出发时间。该时间主要用于：
1.  **计划制定**: 明确行程开始时间。
2.  **夜间行车判断**: 系统根据 "出发时间 + 预计行程耗时" 动态判断是否涉及深夜时段 (02:00 - 05:00)，而非强制基于当前系统时间。

## 2. 完成情况

### 2.1 前端 (Frontend)
- **输入组件**: 在 `RouteForm.vue` 中新增时间选择器组件。
  - 支持日期 (Date) 和时间 (Time) 组合选择。
  - 默认为空 (即使用当前时间)。
  - 支持一键清除。
- **参数传递**: `App.vue` 已更新，将 `departure_time` (ISO 格式字符串) 传递给后端 API。

### 2.2 后端 (Backend)
- **接口更新**: `RoutePlanRequest` 新增可选字段 `departure_time`。
- **逻辑增强**: `check_night_driving` 函数现支持传入 `start_time`。
  - 若用户指定时间，则基于该时间推算行程覆盖时段。
  - 若未指定，则保持原有逻辑 (基于 `datetime.now()`)。

### 2.3 验证结果
- **自动化测试**: `Test/verify_departure_time.py` 验证通过。
  - **Case 1**: 设置出发时间为明日凌晨 03:00 -> 成功检测到 "夜间行车" 标签。
  - **Case 2**: 设置出发时间为明日上午 10:00 -> 未检测到 "夜间行车" 标签 (符合预期)。

## 3. 下一步建议
- **路况预测**: 目前高德 API (标准版) 主要返回当前路况。若未来升级到支持未来时刻路况预测的高级 API，该字段可直接透传给高德以获取更精准的耗时估算。
