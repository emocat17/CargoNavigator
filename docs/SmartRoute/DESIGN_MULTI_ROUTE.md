# Stage 2: 多策略智能选线与数据交付设计文档

## 1. 概述
本阶段目标是实现**多路召回引擎**。系统不再依赖用户选择单一策略，而是后端自动并发请求多种策略（速度优先、费用优先、距离优先等），经过去重和数据增强后，返回 N 条候选路线供用户决策。

## 2. 后端设计 (Backend)

### 2.1 策略定义
我们将同时请求以下高德策略：
- **0 (速度优先)**: 常规最快路线。
- **1 (费用优先)**: 尽量避开收费道路，适合对成本敏感的运输。
- **2 (距离优先)**: 最短路径，往往走国道/省道，可能避开高速限高，但路况复杂。
- **9 (躲避拥堵)**: 实时路况优先。

### 2.2 逻辑流程
1.  **接收请求**: 前端不再传递 `strategy` 参数（或传递默认值）。
2.  **并发请求**: 使用 `asyncio.gather` 同时调用 `AmapService.plan_route_driving` 针对策略 `[0, 1, 2, 9]`。
3.  **结果聚合**: 收集所有返回的 `paths`。
4.  **智能去重**:
    - 依据：`distance` (距离) + `duration` (耗时) + `tolls` (费用) + `traffic_lights` (红绿灯)。
    - 若上述指标完全一致，视为同一条物理路线，仅保留一个副本，但合并其策略标签（如 "速度优先 & 躲避拥堵"）。
5.  **数据增强**: 对每条保留的路线进行 P0/P1/P2 处理（隧道统计、夜间分析、成本估算）。
6.  **响应返回**: 返回 `RoutePlanResponse`，其中 `data['routes']` 包含排序后的 N 条路线。

### 2.3 API 变更
- **Request**: `RoutePlanRequest` 中 `strategy` 字段变为可选或被忽略。
- **Response**: 保持 `RoutePlanResponse` 结构不变，利用现有的 `List[RouteInfo]` 结构返回多条数据。

## 3. 前端设计 (Frontend)

### 3.1 交互变更
- **移除**: `RouteForm.vue` 中的 "选路策略" 下拉框。
- **新增**: 结果面板改为 **路线列表模式**。

### 3.2 组件设计
- **RouteResultPanel.vue**:
  - **模式切换**: 默认显示 "路线概览列表"。
  - **列表项 (RouteCard)**: 展示每条路线的核心指标（耗时、距离、费用、隧道数）及策略标签。
  - **交互**: 点击列表项 -> 高亮地图路线 -> 展开显示详细导航步骤 (Steps)。
